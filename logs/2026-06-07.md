## Pesquisa / aprendizados
- **Serialização e localStorage:** nada que vá para o `localStorage` pode ser função — `JSON.stringify` descarta. Foi a raiz do FIX-003. (Confirmado também que o CLAUDE.md gerado lista só TÍTULO dos prompts — corpo fica na aba Prompts — igual aos nichos prontos.)
- **HTML único vs. modular (discussão, não decidida):** o produto é "1 arquivo, sem build, roda via `file://`/GitHub Pages" (D-001). `fetch()` de JSON quebra via `file://` (CORS); build embutindo JSON reintroduz toolchain. Meio-termo de menor arrependimento: modular no repo + script de concatenação que gera o `index.html` bundled. (i-N13 / ROADMAP Fase 4.)
- **Auto-aplicação de patches / tokens (discussão):** uma ferramenta local que APLICA patches não corta os tokens que o Claude gasta para GERAR; o ganho de token real vem de o Claude **emitir diffs** em vez de arquivos inteiros (i-N16). Patches estilo `apply_patch` precisam de âncoras boas + validação. (i-N15/i-N16 / ROADMAP Fase 5.)

## Tensão aparecida
- A regra dura "entregue o arquivo INTEIRO, nunca trechos" **conflita** com a ideia de "entrega por diff" (i-N16). Reconciliação: diff só quando há uma ferramenta que aplica automaticamente (não o usuário colando à mão). A decidir junto com i-N15.
- O card único do Custom acumula muita coisa numa tela (composer + builder). Em teste ficou aceitável; se crescer, recolher o composer por padrão (anotado em D-019 "a vigiar").
